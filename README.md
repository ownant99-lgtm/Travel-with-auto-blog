# 여행 셀러 블로그 생성기

## 환경변수 설정

1. 프로젝트 루트에서 `.env.example` 파일을 복사해 `.env.local` 파일을 만듭니다.
2. Gemini API Key를 입력합니다.

예시:

```bash
cp .env.example .env.local
```

`.env.local` 예시:

```env
GEMINI_API_KEY=실제_Gemini_API_Key
GEMINI_MODEL=gemini-1.5-flash
```

원하면 `.env` 파일을 직접 만들어도 되지만, 일반적으로는 개인 키를 로컬에만 보관하기 위해 `.env.local` 사용을 권장합니다.

## 현재 동작 방식

- 프론트엔드에서 `URL + 맞춤 가이드라인 4개`를 함께 전송합니다.
- 백엔드에서 타사 페이지의 제목과 본문을 스크래핑합니다.
- 스크래핑한 텍스트는 글감 참고용으로만 사용합니다.
- 실제 블로그 글의 핵심 뼈대는 사용자가 입력한 `코스 순서 / 미팅 정보 / 차별화 혜택 / 가격`을 최우선 반영합니다.
- Gemini `gemini-1.5-flash` 모델에 실제 프롬프트를 전송해 JSON 형식의 장문 블로그 글을 생성합니다.
- 응답이 비어 있거나 너무 짧으면 오류를 반환해 다시 시도할 수 있게 합니다.

## 개발 서버 실행

```bash
npm install
npm run dev
```

환경변수를 새로 추가하거나 수정했다면 개발 서버를 한 번 껐다가 다시 실행해야 반영됩니다.
